# Cross-Market Idiosyncratic Reversion (Hourly)

> **结论：`VALIDATION_FAIL`。本项目是研究代码，不是可上线交易策略。**  
> 验证集全期净收益为正，但前后半段拆分后后半段在基础成本和双倍成本下均亏损，fail-closed 规则生效，测试集保持锁定。

---

## 核心假设

用预声明的经济因子（BTC、ETH、SOL、QQQ、SPY、SOXL）对目标资产做动态 beta 对冲，每个完整小时结束后计算**特质收益**（Idiosyncratic Return）——即剥离系统性市场因子后的残差收益。假设该残差的短期冲击在 2–24 小时内均值回归，并以 Binance USD-M 永续合约为唯一可交易腿捕捉反转机会。

信号生成、成本门槛和 XGBoost 筛选全部使用信号形成前的已知数据；执行时延、价差、手续费、滑点、冲击成本、资金费和资金占用均被显式建模。

## 最终策略

| 维度 | 配置 |
|---|---|
| 可交易腿 | Binance USD-M 永续合约 |
| 目标资产 | 13 个加密资产（crypto_asset 类）|
| 因子腿 | BTC、ETH、SOL、QQQ、SPY、SOXL |
| 数据 | 30 分钟 Binance index/mark bar → 重建小时信号；5 分钟 perp open 精确延迟成交 |
| 动态 beta | RLS/Kalman 形式，半衰期 720 小时；当前小时收益仅在信号形成后更新下一小时 beta |
| 信号 | 最近 2 个完整小时的特质收益冲击 z-score，均值/波动均只使用此前数据 |
| XGBoost | 固定参数，按月扩展训练，仅预测假设方向上的毛反转收益；TreeSHAP 仅作验证诊断，不参与特征筛选或调参 |
| 执行时延 | 决策后 60 分钟成交，120 分钟为压力情景 |
| 主要持有期 | 24 小时 |
| 准入门槛 | 预测毛收益必须覆盖双倍交易成本、资金占用成本及 5 bps 安全边际 |
| 组合约束 | 同标的不重叠持仓；最多 50 组并发；总杠杆 ≤ 1.5x；单品种净敞口 ≤ 5%；同工具目标/因子订单先净额化再计成本 |
| 研究截止 | 2026-07-10（固定环境 Python 3.13.14 + XGBoost 3.3.0）|

## 最终结果

数据集划分（按日期顺序 50% / 25% / 25%，跨边界清除 624 行）：

| 分区 | 日期 / 规模 | 标签行数 |
|---|---|---:|
| Train | 627 个观测日 | 182,482 |

ARB、SUI、JTO 从各自上线日期起进入样本，未补填上线前数据。Binance API 无法恢复已下架合约，存在存续偏差限制。

**验证集结果（notional per leg = $100,000）：**

| 分段 | 交易数 | 毛收益 | 资金费 | 净额交易成本 | 资金占用 | **净收益** | **2x 成本净收益** | 成本覆盖倍数 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 全期 | 203 | $152,234 | $1,851 | $63,205 | $1,504 | **+$89,377** | **+$26,172** | 2.41x |
| 前半段 | 150 | $147,334 | $1,176 | $47,333 | $1,084 | **+$100,092** | **+$52,758** | 3.12x |

后半段失败触发 fail-closed 规则。测试集未用于训练、选模或结果展示。

## 盈亏本质

### 1. 残差平稳 ≠ 累计价差可均值回归

13/13 目标的小时残差收益通过平稳性检验。但累计残差价差的平稳情况：train 2/13、validation 5/13、test 3/13。AR(1) 中位系数约 **0.998**——几乎是随机游走。策略缺少稳定的价格锚，极端偏离更可能是新信息而非暂时错价。

### 2. 未筛选核心净边际为负

验证期 2 小时核心反转均值约 **19.4 bps**，估算净额成本约 **28.6 bps**。XGBoost 在前半段识别出高收益状态，但该条件关系未延续到后半段。

### 3. 尾部延续压低总体期望

核心样本中位毛收益约 34.2 bps，均值仅约 19.4 bps；最差 1% 交易合计约 **-24,718 bps**。少量信息驱动的趋势延续显著侵蚀均值回归的总体期望。

### 4. 亏损不是成本记账误差

归因中：index residual 贡献约 +18.53 bps，perp/index basis 约 +0.04 bps，额外 5 分钟成交差约 +0.82 bps。主要问题是经济信号本身不稳定，而非成本建模错误。

### 5. 更慢 z-score 窗口未修复假设

同一 60 分钟延迟下，未筛选毛收益：2h **17.46 bps** → 4h **3.64 bps** → 8h **-9.76 bps** → 12h **-7.62 bps** → 24h **-26.66 bps**。长窗口把短期反转与中期趋势混合，未降低有效噪声。

### 6. SHAP 与排列重要性的分离

TreeSHAP 在两期高度稳定（Top-5 完全相同，time-based 特征合计占比约 22%）。排列重要性剧烈变化（`boll_width` / `mfi14` 从 2024 无关紧要跃升为 2025–26 最重要特征）。**模型关注 ≠ 信号有效性**；XGBoost 捕捉到了一个真实但不稳定的市场状态。

## 构建记录

1. 将原日频项目完整隔离到 `/hourly`，避免改动原有实现。
2. 构建 L1 采集与严格时间戳校验，排除 Yahoo 小时数据、未来最近邻匹配和不可成交中间价。
3. 历史预筛改用 Binance mark/index bar、实际资金费及下一可成交 bar；加入完整成本链。
4. 从 cash/perp basis 扩展为预声明经济因子下的特质收益策略，加入 30 个目标的因子映射及失效条件。
5. 修正小时边界、UTC/美股 RTH、上线时间、缺 bar、延迟成交、止损和持仓时间计算。
6. 加入动态 RLS/Kalman beta、固定 XGBoost、原生 TreeSHAP、组合敞口限制和订单净额计费。
7. 加入按日期 purge 的 train/validation/test、验证前后半段门槛、测试锁定及含信号/成本参数的缓存指纹。
8. 对比 2/4/8/12/24 小时及日线 z-score、慢动量、混合状态、动态退出；均未稳健改善后半段，保留 2 小时信号。
9. 固定 XGBoost 3.3.0 环境独立重复两次，最终数值一致；29 项因果/成本/分区/组合自动化测试全部通过。

## 运行

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m pytest -q
python main.py tradable-research
```

`tradable-research` 仅使用 Binance 公共历史接口，无需 Alpaca 账号。首次运行将下载约 2023-01-01 至 2026-07-10 的数据并在 `data/` 建立本地缓存（大文件不进 Git）。

**可选命令（需将 `.env.example` 复制为 `.env` 并填写 Alpaca 凭据）：**

```powershell
# 采集 Binance 因子/目标腿 L1，用于重新估计真实价差（不改变历史结果）
python main.py collect-factor-live --duration-hours 8
python main.py analyze-factor-live --raw data/factor_raw
```

## 后续改进

1. **真正前向验证**：从从未查看过的 2026-07-11 之后数据做一次完整前向验证；禁止继续以当前 validation/test 筛参数。
2. **实测成本校准**：用持续采集的 Binance L1 bid/ask 和真实成交回报校准价差、冲击、延迟分布，再以预先固定规则重跑。
3. **升级研究假设**：从"残差收益反转"升级为有经济锚的误差修正/协整价差；只保留跨时期稳定且有可解释锚的配对。
4. **预声明状态过滤器**：信息冲击/趋势延续过滤（市场级跳跃、成交量突变、funding/basis regime），必须先写规则再在新样本检验，不允许回看式删亏损。
5. **存续偏差量化**：引入历史合约主表和已下架合约数据，量化存续偏差；用另一交易所独立复现。

> **任何改进都应遵循同一顺序**：先写经济假设和固定规则 → 锁定成本与验证窗口 → 只查看一次未见数据。当前版本的正确行动是继续研究，不是上线资金。
